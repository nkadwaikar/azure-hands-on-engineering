# Modern Workplace (Microsoft 365): Scenario‑Driven Lab Construction

Identity‑First Security • Zero Trust • Collaboration • Compliance • Governance

This repository contains a complete, enterprise‑grade Modern Workplace engineering track built for Microsoft 365, Entra ID, Exchange Online, SharePoint Online, Teams, Purview Compliance, and Identity Governance.

It reflects real-world patterns used by Senior Cloud & Identity Engineers delivering secure, scalable, governed Microsoft 365 environments in regulated enterprises.

---

## 📌 Track Overview

This track includes **six advanced labs**, each designed to be hands-on, production-aligned, and portfolio-ready:

### 1. **Exchange Online Advanced**  

Enterprise mail flow, transport rules, journaling, anti-phishing, mailbox governance.  
➡️ [1-exchange-online-advanced.md](1-exchange-online-advanced.md)

### 2. **SharePoint Information Architecture**  

Metadata, content types, hub sites, permission models, retention governance.  
➡️ [2-sharepoint-information-architecture.md](2-sharepoint-information-architecture.md)

### 3. **Teams Lifecycle Governance**  

Teams creation governance, naming policies, templates, channel architecture, lifecycle management.  
➡️ [3-teams-lifecycle-governance.md](3-teams-lifecycle-governance.md)

### 4. **Compliance Automation (Purview)**  

DLP automation, auto-labeling, insider risk, compliance manager workflows.  
➡️ [4-compliance-automation.md](4-compliance-automation.md)

### 5. **Zero Trust Advanced (Conditional Access)**  

Device risk, sign-in risk, session controls, passwordless, phishing-resistant MFA.  
➡️ [5-zero-trust-advanced.md](5-zero-trust-advanced.md)

### 6. **Identity Governance (Lifecycle Workflows)**  

Joiner–Mover–Leaver automation, access reviews, entitlement management, deprovisioning workflows.  
➡️ [6-identity-governance-lifecycle-workflows.md](6-identity-governance-lifecycle-workflows.md)

## 🏗️ Architecture Philosophy

This track follows an **Identity‑First, Zero Trust, Governance‑Driven** engineering model:

- **Identity is the control plane**  
- **Least privilege is mandatory**  
- **Conditional Access enforces Zero Trust**  
- **Compliance is automated, not manual**  
- **Teams & SharePoint follow lifecycle governance**  
- **Exchange Online follows secure mail flow patterns**  
- **All workloads integrate with Purview & Entra Governance**  

## 🧭 How to Use This Track

- **Learners:** Build a complete, secure, governed Microsoft 365 environment.  
- **Teams:** Use as a baseline for Modern Workplace modernization.  

Each lab is standalone but designed to be completed in sequence.
